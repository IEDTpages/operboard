# Обновление Оперборда на GitHub Pages — версия v8.2

## Какие файлы заменить

Рекомендуется загрузить в корень репозитория всё содержимое архива с заменой существующих файлов.

Минимально необходимо заменить:

- `index.html`;
- `refresh_data.py`;
- `data/current.json`;
- `data/snapshot.json`;
- `.github/workflows/pages.yml`;
- `tests/test_refresh_data.py`.

`requirements.txt` также следует заменить файлом из комплекта, чтобы версия Playwright оставалась согласованной с Chromium. R и `fedstatAPIr` больше не требуются: ЕМИСС используется только как источник уже импортированной ретроспективы, а новые месяцы загружаются из официальных книг Росстата.

Если в репозитории остались файлы предыдущей версии `tools/fetch_fedstat.R` и `data/fedstat_filters.json`, их можно удалить: новый workflow к ним не обращается.

## Добавление токена ATI.SU

1. Откройте `Settings → Secrets and variables → Actions`.
2. Нажмите `New repository secret`.
3. В поле имени укажите строго `ATI_API_TOKEN`.
4. Вставьте токен ATI.SU в поле значения и сохраните secret.

Не добавляйте токен в `index.html`, JSON-файлы, `refresh_data.py`, переменные GitHub Pages или URL. Workflow передаёт secret только процессу серверного обновления. Браузер посетителя никогда не обращается к API ATI.SU напрямую.

## После загрузки

1. Сделайте commit в ветку `main`.
2. Откройте `Settings → Actions → General`.
3. Проверьте `Workflow permissions → Read and write permissions`.
4. Откройте `Settings → Pages` и проверьте `Source → GitHub Actions`.
5. Добавьте `ATI_API_TOKEN` по инструкции выше.
6. Запустите `Actions → Refresh data and deploy dashboard → Run workflow`.

Ручной запуск сразу проверит обе таблицы Росстата, экспорт и импорт, ERAI, пять рядов Drewry WCI, макроэкономический опрос Банка России, остальные источники и историю общего индекса ATI.SU FTL. Если ATI secret ещё не добавлен, остальные показатели всё равно обновятся.

Если Росстат временно недоступен, workflow сохраняет импортированную ретроспективу и последнюю успешно полученную точку. В логе это отражается как `ERROR` конкретного источника, но корректные существующие ряды не очищаются.

В v7.10 используются актуальные книги `PerevGruz_05-2026.xlsx` и `ind_baza_2023_05-2026.xlsx`. Парсер автоперевозок читает лист `4`, диапазоны `C4:N4`, `B32:B34` и `C32:N34`. Парсер ИПП читает листы `1–3`, строки 4–5 с периодами и строки 6, 7 и 21 с показателями; годы и месяцы могут храниться как текст, числа или даты Excel. При следующих публикациях загрузчик сначала проверяет недавние имена `PerevGruz_MM-YYYY.xlsx` и `ind_baza_2023_MM-YYYY.xlsx`, а затем ссылки со страниц Росстата. Соседняя железнодорожная книга `Pogruzka_MM-YYYY.xlsx` и годовая книга `avto-perev_2025.xlsx` не допускаются к парсеру автоперевозок. Неполная цепочка TLS-сертификатов обрабатывается адресным резервным режимом только для HTTPS-адресов `rosstat.gov.ru`. Отключить его можно переменной `ROSSTAT_TLS_FALLBACK=0`.

## Проверка новых данных

На шаге обновления должны появиться строки:

```text
[refresh] production_index: loading Rosstat workbook…
[refresh] road_freight: loading Rosstat workbook…
[refresh] exports: OK; ...
[refresh] imports: OK; ...
[refresh] ati_ftl: OK; ...
[refresh] erai_composite: OK; ...
[refresh] wci_composite: OK; ...
[refresh] macro_survey_cpi: OK; ...
```

Для каждого источника Росстата результат будет `OK` либо безопасный `ERROR` с сохранением последней корректной истории. При автоматическом запуске возможен статус `skipped`, если таблица уже проверялась менее 20 часов назад.

На шаге `Validate generated JSON` проверяется непустая ретроспектива автоперевозок, а также все три режима ИПП и три линии в каждом режиме.

После публикации на главной странице рядом с ATI.SU должны отображаться карточки `ERAI` и `WCI Drewry`. В навигации у них есть отдельные страницы. На странице внешней торговли доступен переключатель `Месяцы / Годы`, а шесть тематических страниц содержат нижний прогнозный график Банка России.

## Автоматическое обновление

Workflow проверяет общую свежесть данных трижды в час:

```yaml
- cron: "13,33,53 * * * *"
```

Полный сбор запускается только при возрасте данных 50 минут и более. ИПП и автоперевозки внутри полного сбора проверяются не чаще одного раза в 20 часов, поскольку Росстат обновляет их значительно реже. Ручной `Run workflow` принудительно проверяет Росстат независимо от времени предыдущего запроса.
