# Оперборд — GitHub Pages v7.8

Статическая HTML-версия отчёта с интерактивными графиками и автоматическим обновлением данных через GitHub Actions.

## Изменения версии v7.8

- автоперевозки читаются из фиксированного диапазона листа `4`: месяцы `C4:N4`, годы `B32:B34`, значения `C32:N34`;
- ИПП читается из официальных листов `1`, `2` и `3`: годы — строка 4, месяцы — строка 5, совокупный индекс — строка 6, добыча — строка 7, обрабатывающие производства — строка 21;
- надстрочная сноска `¹` в названиях месяцев удаляется до распознавания;
- неполный текущий год обрабатывается корректно: пустые будущие месяцы пропускаются;
- эвристический разбор сохранён только для прежних архивных форматов, где нет официальных листов `1–4`;
- добавлены тесты фиксированных ячеек и сохранения ретроспективы ЕМИСС при слиянии.

Сохраняются изменения v7.7:

- найденный XLSX теперь разбирается сразу: ошибка реальной книги больше не затирается последующими пробами несуществующих выпусков и `404`;
- ИПП поддерживает обе реальные ориентации таблицы Росстата — месяцы по столбцам и месяцы по строкам;
- для автоперевозок поддержаны накопленные выпуски `январь–месяц`: отдельный месяц рассчитывается как разность двух соседних официальных выпусков;
- диагностическое сообщение указывает точное имя скачанного файла и размеры его листов;
- устранены предупреждения `Parsing dates...` в GitHub Actions.

Сохраняются изменения v7.6:

- обновлены прямые адреса Росстата: `PerevGruz_05-2026.xlsx` и `ind_baza_2023_05-2026.xlsx`;
- исправлено название таблицы автоперевозок: «Перевезено грузов автомобильным транспортом»;
- добавлен поиск новых ежемесячных выпусков по шаблонам `PerevGruz_MM-YYYY.xlsx` и `ind_baza_2023_MM-YYYY.xlsx`, если страница каталога временно недоступна;
- парсер автоперевозок теперь обрабатывает как полную месячную таблицу, так и выпуск с одним новым месяцем и отдельной накопленной колонкой;
- накопленные периоды вида «январь–май» не подменяют значение отдельного месяца.

Сохраняются изменения v7.5:

- исправлен сбой `CERTIFICATE_VERIFY_FAILED` при обращении GitHub Actions к `rosstat.gov.ru`;
- при неполной цепочке сертификатов включается ограниченный резервный TLS-режим только для официального домена Росстата;
- каждое перенаправление проверяется: переход за пределы `rosstat.gov.ru` блокируется, а остальные веб-источники по-прежнему используют обязательную проверку сертификата;
- добавлены тесты TLS-fallback, изоляции домена и защиты от внешнего перенаправления.

Сохраняются изменения v7.4:

- ретроспектива автоперевозок за январь 2021 — декабрь 2025 года импортирована из предоставленной выгрузки ЕМИСС 31314;

- при временной недоступности Росстата сохраняется последняя корректная история; автоматическая проверка этих низкочастотных источников выполняется не чаще одного раза в 20 часов;
- на страницах показателей отображаются источник ретроспективы и методологические примечания.

## Источники ИПП и автоперевозок

- ИПП: [официальный раздел Росстата «Промышленное производство»](https://rosstat.gov.ru/enterprise_industrial), таблица «Индексы производства по отдельным видам экономической деятельности по Российской Федерации».
- Автоперевозки: [официальный раздел Росстата «Транспорт»](https://rosstat.gov.ru/statistics/transport), ежемесячная таблица «Перевезено грузов автомобильным транспортом (с 2000 г.)».

Стартовая история остаётся в `data/current.json` и `data/snapshot.json`. Скрипт `tools/import_emiss_history.py` предназначен для воспроизводимого повторного импорта исходных выгрузок ЕМИСС. С 2023 года ряд автоперевозок в предоставленной выгрузке относится к Российской Федерации без учёта ДНР, ЛНР, Запорожской и Херсонской областей; это примечание показано в интерфейсе.

## ATI.SU

Значение токена не хранится в HTML, JSON или Python-коде. Сборщик читает его только из переменной окружения `ATI_API_TOKEN`, которую workflow получает из GitHub Actions Secret с тем же именем. Без секрета остальные источники продолжают обновляться, а карточка ATI.SU показывает состояние ожидания.

## Архитектура обновления

1. `.github/workflows/pages.yml` проверяет свежесть опубликованных данных.
2. `refresh_data.py` получает общедоступные показатели из ЦБ РФ, Росстата, Мосбиржи, ProFinance и IMF PortWatch. Для ProFinance используется Chromium через Playwright.
3. ИПП и автоперевозки обновляются из официальных книг Росстата. Новые точки проходят проверку формата, диапазона, дат и полноты рядов, после чего сливаются с ретроспективой.
4. История ATI.SU запрашивается только серверным workflow с GitHub Secret `ATI_API_TOKEN`.
5. Свежие данные атомарно сохраняются в `data/current.json` и публикуются в GitHub Pages.
6. Если источник временно недоступен или прислал некорректную книгу, его последняя корректная история не стирается.

Ручной запуск workflow принудительно проверяет Росстат. При запусках по расписанию книги Росстата проверяются не чаще одного раза в 20 часов. Для закрытого контура можно установить `ROSSTAT_REQUIRED=1`, чтобы отсутствие обязательного ряда блокировало публикацию; обычный режим сохраняет последнюю корректную версию.

При изменении названия файла на сервере сборщик сначала использует семантический поиск ссылки на странице Росстата. Для диагностики можно временно задать полный URL книги через `ROSSTAT_PRODUCTION_XLSX_URL` или `ROSSTAT_ROAD_XLSX_URL`; секреты для этих источников не требуются.

## Основные файлы

- `index.html` — статический интерактивный дашборд;
- `refresh_data.py` — загрузка, проверка и слияние данных;
- `requirements.txt` — Python-зависимости;
- `.github/workflows/pages.yml` — расписание, Chromium, обновление и публикация;
- `data/current.json` — последний опубликованный набор данных;
- `data/snapshot.json` — резервный встроенный набор данных;
- `tools/import_emiss_history.py` — импорт предоставленных выгрузок ЕМИСС;
- `tools/sync_embedded_data.py` — синхронизация резервного JSON внутри `index.html`;
- `tests/test_refresh_data.py` — тесты парсеров, слияния и безопасного ATI-подключения.

Инструкция по замене файлов находится в `GITHUB_PAGES_SETUP.md`.
